# 知宠官网 Codex 开发规则

## 项目目标

开发一个基于 Nuxt 4、Vue 3 和 TypeScript 的知宠企业官网。

## 核心原则

1. 不编造产品、公司、客户、团队、地址、电话、资质和数据。
2. 缺失内容使用 TODO 或“待项目方确认”标记。
3. AI 生成素材必须标注“AI 概念图，仅供参考”。
4. 优先复用组件，不复制页面代码。
5. 使用 TypeScript，禁止 any。
6. 保持页面克制，避免卡片过密。
7. 不安装后台型 UI 组件库。
8. 不加入 Three.js、粒子和强制平滑滚动。
9. 每个页面必须响应式。
10. 每次修改后运行 lint、typecheck 和相关测试。
11. 未经允许不修改无关文件。
12. 未经允许不自动提交 Git。

## 长期约束

1. 不得把 AI 宠物理解描述成宠物语言逐字翻译。
2. 所有 AI 判断均应使用“可能”“推测”“趋势”“风险提示”等概率表达。
3. AI 分析展示必须包含依据和不确定性。
4. 未经确认，不得声明设备具有医疗级生命体征能力。
5. 原始音频、家庭环境音频和宠物数据均视为敏感数据。
6. 不得编造模型准确率、数据规模和合作机构。
7. 新功能必须复用现有设计系统，不得另起一套页面风格。
8. 本轮修改属于增量升级，禁止无关重构。

## 设计规则

- 最大内容宽度 1200px。
- 使用暖白、深咖和金棕色。
- 普通卡片默认无阴影。
- 每个模块最多一个主按钮。
- 每个内页控制在 5–7 个核心模块。
- 模块上下间距桌面端不少于 96px。
- 页面必须支持 prefers-reduced-motion。
