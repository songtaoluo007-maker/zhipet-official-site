# 知宠 ZHIPET 官网规范

## 项目定位

知宠 ZHIPET 官网用于对外说明品牌定位、产品方向、使用边界与交流入口。它不是电商商城，也不是 SaaS 管理后台。

核心目标：

1. 清楚解释知宠是什么。
2. 展示产品、解决方案和技术能力。
3. 服务家庭用户、宠物医院、宠物门店和合作机构。
4. 通过真实内容、清晰边界和可核验资料建立可信度。
5. 只通过公开邮箱和意见问题反馈承接对外联系，不在官网设置用户登录。

## 内容原则

- 禁止编造产品、公司、客户、团队、地址、电话、备案、资质和数据。
- 缺失或未核实内容不在公开页面展示，也不使用暴露内部状态的占位文案。
- 不展示未核实的公司名称、电话、地址、资质或团队个人信息。
- AI 生成图片必须标注“AI 概念图，仅供参考”。
- 团队人像、证书、二维码、电话、地址、备案号不得生成为看似真实的凭证。

## AI 宠物理解产品主线

知宠的核心差异化能力不是对宠物语言进行逐字翻译，而是：

通过宠物声音、动作姿态、活动节律、场景环境和长期个体数据，对宠物可能的情绪、需求、行为意图和异常状态进行多模态概率分析，并向主人提供可理解的分析依据、置信度、其他可能解释和照护建议。

产品体系调整为：

1. 知宠智能挂件 / 感知设备
2. 知宠 App
3. PetSense 多模态宠物理解引擎
4. 知宠专业服务平台

AI 宠物理解结果必须体现：

- 当前可能状态
- 分析依据
- 置信度
- 其他可能解释
- 主人建议
- 风险等级
- 数据不足或不确定性说明
- 医疗边界说明

禁止宣传：

- 百分之百听懂宠物
- 宠物语言逐字翻译
- 人宠无障碍对话
- 医疗级诊断
- 未经验证的准确率
- 未经验证的疾病识别能力

新增页面路由：

`/ai-pet-understanding`

首页新增“AI 宠物理解引擎”模块。

家庭养宠解决方案增加：

- 宠物情绪理解
- 异常叫声分析
- 行为变化识别
- 个体化陪伴建议
- 长期宠物画像

## 视觉原则

关键词：温暖、专业、可信、克制、科技感、宠物友好。

禁止：满屏科技蓝、霓虹渐变、过量玻璃拟态、大量悬浮小卡片、无意义粒子动画、AI 后台感组件、虚假夸张数据。

## 组件原则

使用轻量官网 UI Kit：按钮、链接、标签、图片、容器、表单输入、导航、页脚、页面 Hero、Section Heading 和 CTA。

## 动效原则

允许轻微动效：导航吸顶、按钮反馈、图片淡入、模块 reveal、移动菜单过渡。禁止 Three.js、粒子、强制平滑滚动、大范围视差和长加载动画。必须支持 `prefers-reduced-motion`。
